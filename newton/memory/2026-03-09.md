#### 关键 API 端点

```
# Vanguard 协调
GET  /api/requirements?status=new          # 待办需求
GET  /api/teams/online                     # 在线团队
POST /api/requirements/<id>/assign         # 分配需求

# 团队任务
GET  /api/teams/<team>/assigned-requirements
POST /api/requirements/<id>/take           # 领取需求
POST /api/teams/<team>/machine-status      # 上报机器状态
POST /api/teams/<team>/status-report       # 上报状态（含tasks）

# Hera/Vanguard 汇总
GET  /api/dashboard/risk-report
GET  /api/teams/status-report/summary
POST /api/feishu/post                      # 发飞书
```

### 汇报规范（重要）

**每次汇报必须包含：**
- 机器状态：CPU、内存、磁盘、负载
- 队员状态：每位 Agent 的角色和状态
- 需求进度：总需求数、进行中、完成数
- **队员任务**：每位成员分配到的具体任务和进度
- 子任务状态

- ✅ 需求领取功能正常
- ✅ 状态上报功能正常
- ✅ 24h 机器状态报告测试进行中
